# Data

Run `python scripts/fetch_data.py --all` to populate this tree. What's tracked
in git vs. fetched on demand:

```
data/
  hte/          TRACKED — small, essential HTE yield data (the SFT/RL targets)
    Buchwald-Hartwig/Dreher_and_Doyle_input_data.xlsx   (~2.1 MB, vendored)
    Suzuki-Miyaura/...                                   (optional transfer ablation)
  raw/          GITIGNORED — large pretraining corpora (fetched, then tarballed)
    pubchem.smi            (Stage 1 molecular grammar)
    uspto_*.rsmi.gz        (Stage 2 reactivity)
    ord/                   (Stage 2 reactivity, schema-typed)
  processed/    GITIGNORED — generated by scripts/prepare_corpus.py
    tokenizer.json, pubchem.smi (cleaned), reactions.* (pretokenized mmap)
```

## Sources & licenses

| Dataset | Source | Size | Reachable from Claude sandbox |
|---------|--------|------|-------------------------------|
| **Buchwald-Hartwig** (primary) | rxn4chemistry/rxn_yields (GitHub) — Ahneman et al., *Science* 2018 (Doyle/Merck) | ~2 MB | ✅ vendored |
| **Suzuki-Miyaura** (secondary) | Perera et al., *Science* 2018 / rxn4chemistry splits | ~few MB | ⚠️ path not stable; fetch by hand |
| **PubChem** (Stage 1) | NCBI FTP `CID-SMILES.gz` | ~2 GB (use 5–10M) | ❌ blocked here — fetch on the GPU box |
| **USPTO / Lowe** (Stage 2) | figshare 5104873 (1976–Sep2016) | ~1.5 GB | ❌ blocked here — fetch on the GPU box |
| **Open Reaction Database** (Stage 2) | open-reaction-database/ord-data (GitHub) | large (sharded protobuf) | ✅ reachable via raw |

The Buchwald-Hartwig data is public-domain from the Doyle/Merck HTE campaign,
redistributed by IBM's rxn4chemistry. Reaxys/CAS/Pistachio are **not** used —
their terms prohibit training (design §2).

## Notes

- **PubChem and USPTO are blocked by this sandbox's proxy**, so they can't be
  pulled into the repo/tarball from a Claude session. Run
  `python scripts/fetch_data.py --pubchem --limit 10000000 --uspto` on the H100
  box (open network), then `scripts/prepare_corpus.py` to clean + pre-tokenize.
- The BH `.xlsx` has 16 sheets: `FullCV_01..10` (canonical random folds) and
  `Test1..4` (additive-holdout OOD). `data/dataset.load_buchwald_hartwig` reads
  them; verified to load 3,955 reactions from `FullCV_01`.
- Turning raw USPTO/ORD into the slot-tagged `reactions.jsonl` that
  `prepare_corpus.py` expects (heuristic agent-role tagging for USPTO, schema
  roles for ORD) is the one piece of upstream parsing still to write — see
  `docs/PIPELINE.md`.
